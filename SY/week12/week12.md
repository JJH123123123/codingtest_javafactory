## 문제 해설 

Problem 245: 수 찾기<br>
-O(N)<br>
-입력값을 기록하고 새로들어오는 쿼리에대해 찾는다, 해시로 으로 조회시 O(1)로찾는다<br>

Problem 246: 숫자 카드 2<br>
-O(N)<br>
-입력값을 기록하고 새로들어오는 쿼리에대해 찾는다, 해시로 으로 조회시 O(1)로찾는다<br>

Problem 247: 좌표 압축<br>
-O(NlogN)<br>
-좌표를 압축시키기위해 입력된 배열을 별도로 정렬하기위한 배열에 추가로 기록후 정렬한다, 이후 정렬된 순서를 기준으로 자신의 앞에있는 순서를 기반으로 갯수를 기록하는 변수를 이용하여 앞에 위치한 갯수를 기록하고 이를 사용해 압축된 결과를 출력한다<br>

Problem 248: 세 수의 합<br>
-O(N^2logN)<br>
-입력값이 1000개이므로 이를n^2 * log N으로 해결하기위해 두개를 더한 조합의합을 구해놓고 이를 기준으로 세 수의 합이 배열내에 존재하는 값이기 때문에 투포인터를 이용해 큰값-작은값으로 투포인터를 기준으로 찾되 같은값이 존재하는가 이분탐색으로 logN으로 빠른탐색하는데
X + Y + Z = K이므로 X + Y == K - Z를 양측비교하여 투포인터로 i,j지정후 mid를 이분탐색하여 찾는것<br>

Problem 249: 랜선 자르기<br>
-O(N + logN)<br>
-t/f결정문제에서 최대t를 이분탐색으로 찾는다<br>

Problem 250: 숫자 카드<br>
-O(NlogN)<br>
-입력값에 존재하는가 이분탐색을 이용해 찾는다<br>

Problem 251: 차집합<br>
-O(N)<br>
-값을 입력받고 정렬후, 원소를 순회하면서 기록하는 변수를 이용하여 원소가 겹치지 않는 값을 찾아야하기에 이전값까지 반복문을 이용해 증감시켜가며 집어넣는다, 
원소가같으면 삽입하지않고 인덱스만증감시켜 차집합을 찾는다<br>

Problem 252: 과자 나눠주기<br>
-O(N + logN)<br>
-이분탐색을 이용해 t/f중 최대t를 찾는다, 결정문제는 찾는값으로 나눠서 갯수를세어 정한다<br>

Problem 253: 나무 자르기<br>
-O(N + logN)<br>
-결정문제 t/f중 최대t를 찾는다, 결정문제는 찾는값으로 찾는값으로 차를 더해 나무의 합을 기준으로 정한다<br>

Problem 254: 멀티버스 Ⅱ<br>
-O(NlogN)<br>
-조건대로 같은 패턴을 찾는 좌표압축문제, 정렬후 중복원소를 제거한 길이만큼 탐색하여 같은좌표압축결과인지 이분탐색을 이용해 압축결과를 기록하고 이를 조합을 이용해 같은갯수를센다<br>

Problem 255: 용액<br>
-O(NlogN)<br>
-정렬후 투포인터를 이용해 더한값이 0보다 크면 r--, 작으면 l++하여 값을0에맞게끔 찾는다, 이분탐색의경우 전체순회하면서 나머지한쌍을 이분탐색으로 최소값을찾는다<br>

Problem 256: 합이 0<br>
-O(N^2logN)<br>
-정렬후 조합을 이용해 n^2꼴수를 찾고 나머지 수 하나를 중복수를 카운팅하여 갯수를센다, ub-lb로 같은수 쌍을 세어 더한다<br>

Problem 257: 용액 합성하기<br>
-O(NlogN)<br>
-용액문제와같음<br>

Problem 258: 좋다<br>
-O(N^2logN)<br>
-정렬후 조합을 이용해 n^2꼴수를 찾고 나머지 수 하나를 중복수를 카운팅하여 갯수를센다, ub-lb로 같은수 쌍을 세어 더한다, 투포인터 혹은 이분탐색으로센다 합이0문제와마찬가지<br>

Problem 259: 두 배열의 합<br>
-O(N^2logN)<br>
-a배열, b배열의 누적합 형태의 배열을 원소로 사용하는 누적합배열을 두개 조합으로 만들고 이를 이분탐색, 혹은 투포인터를 이용하여 값을찾는다, ub-lb로 중복수를 경우의수곱으로 더한다 <br>

Problem 260: 세 용액<br>
-O(N^2 + N ~ N^2logN)<br>
-용액 두개에서 한가지 경우의수가 더해졌으므로 n^2을 찾고 이후 투포인터 혹은 이분탐색으로 나머지 수 하나를 찾는다<br>

Problem 261: 공유기 설치<br>
-O(N + logN)<br>
-이분탐색을 이용해 t/f결정문제중 최대t를 찾는다, 회의실배정 문제처럼 공유기의 설치위치pos를 기준으로 공유기설치의 수를 결정문제로 사용해 위치를 기록하며센다<br>

Problem 262: 합이 0인 네 정수<br>
-O(N^2*logN)<br>
-배열 4개를 배열0,1 조합, 2,3조합으로 더한 조합수를 기준으로 양측 합배열을 투포인터 혹은 이분탐색을 이용해 ub-lb를 찾아 더한다<br>

Problem 263: 가장 긴 증가하는 부분 수열 2<br>

Problem 264: 예산<br>

Problem 265: 수 고르기<br>
-O(N + logN)<br>
-이분탐색을 이용해 정렬된값에서 M을 기준으로 양측값을 logN조회하면서 절대값 기준으로 sum이  M<=sum의 최소값을 찾는다, 이분탐색하면서 조건부합시 min을구하며기록.<br>

Problem 266: 부분합<br>
-O(N^2 + N)<br>
-누적합을 원소로 사용하면서 그중 f/t에서 최소t를 이때 누적합중 길이를 찾는것이므로 len을 원소로두고 arr[i] - arr[i-len]이 결정문제 S<=누적합결과 인지 찾아서사용<br>

Problem 267: 소수의 연속합<br>

-소수를 입력값범위까지 구해놓고 투포인터를 이용해 sum보다 작으면 r++, 크면 l++로 합차를 가감산을 해가며 sum갯수를 계수해 갯수를센다<br>

Problem 268: 수들의 합 2<br>

Problem 269: List of Unique Numbers<br>

Problem 270: 가장 긴 짝수 연속한 부분 수열 (large)<br>

Problem 272: 겹치는 건 싫어<br>

Problem 273: 대표 선수<br>

Problem 274: 구간 자르기<br>
